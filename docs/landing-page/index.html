Capabilities

Why AURA?

Built from the ground up as a research project, AURA brings enterprise-grade automation to your desktop.

🎯

Goal-Oriented AI

Understands semantic goals — "open YouTube and search nvidia" is intelligently merged into one optimized action, not two separate steps.

🗣️

Voice & Wake Word

Say "Hey AURA" for hands-free control. Offline wake-word detection with continuous speech recognition built in.

🔧

50+ System Tools

File operations, volume, brightness, screenshots, clipboard, window management, virtual desktops — all deterministic, all built-in.

🛡️

Secure by Design

No exec() or eval(). Schema-validated LLM outputs. Domain-locked tool resolution prevents hallucinated or dangerous actions.

🧠

Multi-LLM Support

Gemini, Ollama (local), OpenRouter. Hybrid mode: local-first for privacy, cloud fallback for reliability. 85-95% API cost reduction.

📱

Remote Control

Telegram bot integration for remote commands. Email via SMTP. Manage your PC from anywhere in the world.

System Design

Three-Layer Architecture

Strict separation between AI reasoning and deterministic execution ensures safety and reproducibility.

⚡ Layer 1 — Routing 0 Tokens (local)

Fast syntactic classification of queries into single or multi-goal categories. Minimal LLM inference for maximum cost efficiency.

GoalRouter SinglePath MultiPath
▼
🧠 Layer 2 — Reasoning ~100 Tokens

Semantic understanding through specialized agents. Extracts goals, builds dependency graphs, and creates minimal executable plans.

IntentAgent GoalInterpreter GoalPlanner GoalOrchestrator
▼
🔒 Layer 3 — Execution Deterministic

Domain-locked tool resolution with precondition enforcement. Every tool satisfies a deterministic contract — same inputs, same results.

ToolResolver ToolExecutor SafetyValidator
Workflow

From Voice to Action

Natural language in, deterministic execution out. Here's how AURA processes a complex command.

🎤
Step 01

Speak or Type

"Hey AURA, create folder reports and put a summary inside it"

🧠
Step 02

Goal Extraction

AURA extracts 2 dependent goals and builds a DAG with correct execution ordering

⚡
Step 03

Smart Execution

Dependency-aware: folder created first, then file placed inside — deterministic and safe

✅
Step 04

Done

"Folder 'reports' created with summary.txt inside. Anything else, sir?"
